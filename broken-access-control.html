Broken Access Control

Description

Access control enforces policy such that users cannot act outside of their intended permissions. Failures typically lead to unauthorized information disclosure, modification, or destruction of all data, or performing a business function outside of the user's limits.

Example: IDOR

An application uses unverified identifiers in an HTTP request to access data:

https://example.com/api/v1/users/1234/profile

By changing `1234` to another ID, an attacker can access other users' private information.

Mitigation

  • Deny by Default: Everything is restricted unless explicitly allowed.
  • Implement access control mechanisms once and re-use them throughout the application.
  • Disable web server directory listing and ensure file metadata (e.g., .git) is not reachable.
