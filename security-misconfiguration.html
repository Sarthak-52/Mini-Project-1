Security Misconfiguration

Description

Security misconfiguration is the most common vulnerability. This is often the result of default configurations or incomplete setups. It can happen at any level of an application stack, including network services, platform, web server, or custom code.

Example: Default Credentials

A web server's admin panel is left accessible with default credentials:

Admin / admin

Attackers scan for these panels and can take full control of the application and its data within seconds.

Mitigation

  • Change all default passwords and security settings immediately upon installation.
  • Harden the server by removing unused features, components, and samples.
  • Review and update configurations periodically.
