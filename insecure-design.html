Insecure Design

Description

Insecure design is a broad category that represents different flaws, including missing or ineffective control design. It is not about implementation; rather, it refers to vulnerabilities that exist because the application's actual design is flawed from the beginning.

Example: Logic Flaw in Checkout

A web store allows users to modify the 'price' field in an HTML form before submitting it to the server:

<input type="hidden" name="price" value="99.99">

If the server doesn't re-validate the price from its own database, an attacker can pay $0.01 for a $99 item.

Mitigation

  • Establish and use a **secure development lifecycle** (SDLC).
  • Design with security in mind (Secure by Design).
  • Never trust client-side data for critical logic like pricing, permissions, or calculations.
