Cryptographic Failures

Description

Formerly known as Sensitive Data Exposure, this occurs when sensitive data (like passwords, credit card numbers, and PII) is not adequately protected from attackers. This can happen due to lack of encryption, weak encryption algorithms, or insecure key management.

Example: Plaintext Storage

A website stores user passwords in a database without hashing them:

db.users.save({ username: "user1", password: "mypassword123" });

If the database is leaked, every user's password is immediately visible to the attacker.

Mitigation

  • Always hash passwords using a strong, salted algorithm like **Argon2** or **bcrypt**.
  • Encrypt all sensitive data in transit (using TLS) and at rest.
  • Avoid using legacy or weak cryptographic functions like MD5 or SHA1.
