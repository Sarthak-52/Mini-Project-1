Injection Attacks

Description

Injection vulnerabilities occur when untrusted data is sent to an interpreter as part of a command or query. The attacker's hostile data can trick the interpreter into executing unintended commands or accessing data without proper authorization.

Example: SQL Injection

An attacker bypasses a login screen by entering a special sequence in the username field:

' OR '1'='1' --

This changes the query logic to always return true, allowing access without a valid password.

Mitigation

  • Use Parameterized Queries (Prepared Statements).
  • Implement Input Validation (Allow-listing).
  • Escape all user-supplied data before using it in queries.
